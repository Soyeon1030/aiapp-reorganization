핵심 원칙 — 규격은 고정, JSON이 값만 채운다

레이아웃은 B2 스튜디오와 동일한 2단(좌 채팅 · 리사이저 · 우 패널). 흐름은 요구사항 입력 → 티키타카(질의) → PRD 일괄 채움: 질의 중 우측은 빈 규격(스켈레톤) — 섹션 제목은 실제, 값 자리만 회색 바. 질의가 끝나면 에이전트 JSON 1회 도착으로 문서 전체가 한 번에 바인딩된다. 중간중간 채우는 연출 없음 — 프런트는 고정 템플릿에 값만 꽂으면 되므로 구현·응답이 빠르다. 문서에 이모지 금지 — 플랫하고 깔끔하게. 완성 후 접힘 재선택으로 답을 바꾸면 그때만 즉시 갱신.

흐름 — 여기까지가 이 화면의 범위
B1 아이디어 전송 한 문장 요청
→
AI ① 기능 질의 2개 답 → 우측 ①에 "대화로 확정" 행
→
AI ② 디자인 질의 2개 답 → 우측 ② 디자인 방향 카드
→
AI → 확인 대기 🔒 기획안 완성 화면 흐름·저장 정보 공개 + 확정
우측 패널(아티팩트) 규칙
  • 배경은 미리보기 페인과 같은 rgb(249 250 251) — 스튜디오와 시각 연속성
  • 질의 중 = 빈 규격(스켈레톤): 섹션 제목 6개는 실제, 값 자리는 회색 바 — "무엇이 만들어질지"의 예고편 역할
  • 질의 완료 = JSON 1회 일괄 바인딩: 스켈레톤이 채워진 문서로 교체(스크롤 맨 위로) · 배지 "대기 중"→"완성" · 확정 버튼 활성
  • 상단 바 상태 문구: "질문이 끝나면 여기에 채워져요" → 완료 시 "확인해 주세요"
  • 완성 후 접힘 재선택으로 답을 바꾸면 해당 슬롯만 즉시 갱신
  • 리사이저는 b2와 동일(기본 채팅 35%, 300~720px 드래그) · 900px 이하 1단
질의 규칙 (채팅 쪽)
  • 되묻기 카드는 제목+설명문(§2.4) · 옵션 2~3 + AI 추천 · 디자인 옵션에 톤 견본 · "잘 모르겠어요" = 추천 적용
  • 질문 수 고정: 기능 2 + 디자인 2 — 화면·데이터·스타일 토큰이 실제로 갈리는 것만
  • 질의 중 컴포저는 타이핑 가능·전송만 잠금 → 기획안 완성 시 해제(수정 = 채팅, 별도 수정 버튼 없음)
  • 기획안 문서에 이모지 금지 — 태그·견본·타이포만으로 구분 (2026-07-22 확정)
기획안 문서 ↔ 에이전트 JSON (Manifest PRD 절충 — 2026-07-22)

Manifest의 기획서 항목을 전부 수용하되, 사장님이 확인할 것은 본문에, 전문가용 서사(지표·리스크)는 접이식으로 나눈다. 기존 JSON에 없는 항목은 확장 필드로 제안 (프롬프트 원문 문서 참조).
