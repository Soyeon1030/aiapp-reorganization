설계안 v2 — 기획서 3탭 노출 (기획 에이전트 출력의 시각화)NEW 07-22 15:00

핵심 원칙 — 티키타카로 구체화, 결과 먼저, 상세는 점진 공개

기획 에이전트의 JSON 하나가 이 화면 전체의 데이터 출처다. 흐름은 두 박자: ① 요청 직후 인터뷰(선택지 티키타카)로 서비스의 갈림길만 빠르게 확정하고 ② 기획안은 문서가 아니라 생성된 사이트 미리보기로 먼저 보여준 뒤 요약 카드로 확인만 받는다. 화면·데이터가 궁금한 사람만 기획서 탭을, 문서가 필요한 사람만 자세히 탭을 연다.

인터뷰 단계 — 질문의 규칙 NEW 07-22 15:13
  • 질문 수 고정 2~3개 — 피로 방지. 기준: 답에 따라 화면 모양이나 저장 정보가 실제로 달라지는 갈림길만 묻는다 (예약 방식·산책러 구성·요금). 문구·색처럼 나중에 바꿔도 되는 건 묻지 않는다
  • 옵션 2~3개 + ✨ AI 추천 표시(§2.4 추천 배지 기준) · 각 옵션에 "고르면 뭐가 달라지는지" 한 줄 설명
  • 전부 스킵 가능 — "잘 모르겠어요"는 추천 적용. 모른다고 흐름이 막히지 않는다
  • 답한 질문은 접힌 기록으로 쌓이고, 클릭하면 그 질문부터 재선택 (blueprint 접힘 문법과 동일)
  • 답은 요약 카드에 "대화로 확정"(초록)으로 승격 — 말씀하신 내용 / 대화로 확정 / 추측함(주황)의 3단 신뢰도가 이 화면의 시각 언어
  • 다 답하면 "정리하고 있어요…" 비트 후 기획안 공개 — 인터뷰 답은 에이전트 입력의 누적 맥락으로 전달

카피 규칙
  • 내 요청을 상단에 그대로 인용 — AI 이해의 기준점을 사용자 말로 고정
  • 개발 용어 금지: 테이블 → "저장 공간", 필드 → "항목". 타입은 쉬운 말 7종(글자/숫자/날짜/전화/선택/이미지/참·거짓)
  • 추측(assumed)은 점선 + "추측함" — "다르면 알려주세요"로 부담 없이 정정 유도
  • 크레딧은 실사용만 표기("무료 생성 1회가 사용돼요") — 견적·예상 수치 금지
  • 미리보기 주소 표시는 출시 게이트 규칙(§2.2)을 따름 — "아직 주소가 없어요"
미정 — 협의 필요
  • 기존 설계안 대화(blueprint ①~⑤)와의 관계: 인터뷰+3탭이 그 자리를 대체하는지, 병행하는지
  • 인터뷰 질문을 누가 뽑나: 기획 에이전트가 요청 분석 후 동적 생성(추천) vs 업종별 고정 세트
  • 미리보기 목업의 충실도: 생성 전 단계에서 어느 수준까지 실제처럼 보여줄지 (기대치 관리)
  • [조금 고칠게요]의 동작: 대화 복귀(현재 시안) vs 요약 카드 인라인 수정
