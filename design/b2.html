AI 응답 치트시트 (명세 §2.2·2.4)

AI는 아래 7가지 형태로만 답해요 — 자유 형식 금지. 카드 속 미리보기가 실제 컴포넌트 그대로입니다.

AI 작업 노출 범위 — 어디까지 보여주나 (2026-07-24 v3)

두 층 + 비노출. 일반 사용자가 보는 기본 표면은 요약만, 원문 스트림은 접힌 "실시간 작업 내용" 패널에, 나머지는 아예 보여주지 않는다. 기준은 개발 문서 "AI 작업 실시간 스트림"과 동일.

정보노출 위치비고
짧은 답변 문장 (안내 · 발견 보고)기본 표면타이핑 누적 — 해요체, 1~2문장
작업 계획 · 현재 수행 (훅)패널 헤더 한 줄 — 지금 하는 일 + 진행률 N/M체크리스트 전체는 패널 안 — 독립 계획 카드 없음 (v3.1)
작업 완료 사실기본 표면 — ✓ 체크 줄사실 + 사용자 이득 한 줄
추론(작업 구상) 원문상세 패널영문 포함 그대로 — 가공 없음
파일별 상태 (경로 · 작업 중/완료/실패)상세 패널경로만 — 내용 없음

중복 훅 이벤트비노출동일 작업 중복 표시 방지
  • 기본 표면 금지어 — 파일 경로 · 영문 원문 · 마크다운 문법(**별표** 등)은 기본 표면에 올리지 않는다 (전부 상세 패널 몫)
  • 훅에 없는 이벤트를 요구하지 않는다 — 그룹 구분선 · 단계 배지 등은 스펙 제외. 새 요소는 타임라인 아래로만 추가
  • 상세 패널은 기본 접힘 — 헤더 한 줄이 지금 하는 일 + 진행률 N/M + 경과 시간을 담당. 펼침 상태는 세션 동안 유지

✍ AI 카피 규칙 5가지
  • 해요체 · 한 메시지 = 한 사실 · 수치와 상태는 카드에, 문장은 짧게
  • 개발 용어 번역: 테이블→예약/회원, SSE→실시간 연결, VM→전용 공간
  • 가능하면 행동 유도로 끝맺기 — "오른쪽 데이터 탭에서 확인하세요"
  • 되돌리기는 "이전으로" (여러 단계 히스토리가 아니므로 '버전' 금지)
  • 이모지는 알림성(🔔 📢)만 최소 사용

질문결정
로그는 스트리밍?줄 단위 완성형 — 타자 효과 없음. 이벤트 1개 = 채팅 요소 1개, 이벤트 type이 위 문법 표와 1:1 (log·step·check·console_update·pause·ask·report·suggest·error). 문구 생성은 전부 서버 책임
수정 요청 처리 표시?짧은 작업은 로그 2줄로 끝: "바꾸고 있어요…" → ✓ 완료 + 미리보기 갱신 + '이전으로' 활성화. 큰 작업만 진행 상태 카드 재사용
되묻기는 언제?①복수 해석 가능 ②필수 값 누락 — 두 경우만. 취향 차이는 묻지 않고 추천으로 진행 후 보고. 요청당 1회, 꼬리 질문 금지
질문 무시하고 딴 말 하면?못 보냄 — 질의 활성 중엔 전송 잠금 (타이핑은 가능, 미리 써두기 OK). 자유 답은 카드의 '기타' 입력으로. 답하면 즉시 해제 (지금 이 화면 하단이 그 상태)
AI 작업 중 입력 가능?타이핑은 가능, 전송만 잠금 — "지금 작업 중이에요, 끝나면 보낼 수 있어요". 완료 시 자동 해제, 써둔 내용 유지. 큐잉 없음
'AI 추천' 배지 기준?질문당 1개. ①프로젝트 맥락 정합(기존 선택·브랜드) → ②동종 서비스 모범 사례 → ③동률이면 더 가역적인 쪽. 근거는 옵션 설명 한 줄로 노출, 근거 없으면 배지 없음
추천 칩("이어서…") 기준?출시 보고·수정 완료 직후에만 1세트(3~4개, 2개 미만이면 생략). 슬롯 순위: ①할 일(보고 카드와 중복 시 생략) ②방금 작업의 후속 ③성장 제안 ④탐색. 문구는 사용자 1인칭 요청문(클릭 = 입력창 채움), 실행·미클릭 2회면 교체
재접속 시 복원 범위?채팅 이력 전체. 카드는 최종 상태(접힘/스킵)로, 미답 차단 카드가 있으면 활성 그대로 + 거기로 스크롤
