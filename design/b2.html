질문결정
로그는 스트리밍?줄 단위 완성형 — 타자 효과 없음. 이벤트 1개 = 채팅 요소 1개, 이벤트 type이 위 문법 표와 1:1 (log·step·check·console_update·pause·ask·report·suggest·error). 문구 생성은 전부 서버 책임
수정 요청 처리 표시?짧은 작업은 로그 2줄로 끝: "바꾸고 있어요…" → ✓ 완료 + 미리보기 갱신 + '이전으로' 활성화. 큰 작업만 진행 상태 카드 재사용
되묻기는 언제?①복수 해석 가능 ②필수 값 누락 — 두 경우만. 취향 차이는 묻지 않고 추천으로 진행 후 보고. 요청당 1회, 꼬리 질문 금지
질문 무시하고 딴 말 하면?못 보냄 — 질의 활성 중엔 전송 잠금 (타이핑은 가능, 미리 써두기 OK). 자유 답은 카드의 '기타' 입력으로. 답하면 즉시 해제 (지금 이 화면 하단이 그 상태)
AI 작업 중 입력 가능?타이핑은 가능, 전송만 잠금 — "지금 작업 중이에요, 끝나면 보낼 수 있어요". 완료 시 자동 해제, 써둔 내용 유지. 큐잉 없음
'AI 추천' 배지 기준?질문당 1개. ①프로젝트 맥락 정합(기존 선택·브랜드) → ②동종 서비스 모범 사례 → ③동률이면 더 가역적인 쪽. 근거는 옵션 설명 한 줄로 노출, 근거 없으면 배지 없음
추천 칩("이어서…") 기준?출시 보고·수정 완료 직후에만 1세트(3~4개, 2개 미만이면 생략). 슬롯 순위: ①할 일(보고 카드와 중복 시 생략) ②방금 작업의 후속 ③성장 제안 ④탐색. 문구는 사용자 1인칭 요청문(클릭 = 입력창 채움), 실행·미클릭 2회면 교체
재접속 시 복원 범위?채팅 이력 전체. 카드는 최종 상태(접힘/스킵)로, 미답 차단 카드가 있으면 활성 그대로 + 거기로 스크롤
