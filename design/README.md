# AiApp 화면 디자인 (정적 HTML/CSS)

`aiapp-prototype.html`(인터랙티브 프로토타입)을 기반으로, 실제 화면을 페이지별 정적 마크업으로 분리한 작업 공간입니다. **UI/UX 전용** — 데모 JS 없이 마크업과 스타일만 다룹니다.

## 구조

```
design/
├── css/
│   ├── tokens.css       # 디자인 토큰 — 컬러·타이포·스페이싱·라디우스·그림자 (원자 값의 유일한 출처)
│   ├── components.css   # 공통 컴포넌트 — 앱 셸(gnb)·버튼·칩·태그·배지·카드·헬프 버튼
│   ├── b1.css           # B1 전용 — 히어로·컴포저(프로젝트 어태치)·작업 유형 칩·프로젝트 카드
│   ├── b2.css           # B2 전용 — 채팅 피드·진행 상태 콘솔·보고 카드·미리보기 패널
│   └── db.css           # 데이터 패널 전용 — 유형 사이드바·인박스 리스트·필터
├── b1.html              # B1 시작 화면
├── b2.html              # B2 스튜디오 (기준 상태: 출시 완료 LIVE, 미리보기 탭)
├── db.html              # B2 스튜디오 — 데이터베이스 탭 (예약 인박스)
└── README.md
```

## 규칙

- **원시 값 금지**: 페이지 CSS에서 hex·px를 직접 쓰지 않고 `tokens.css`의 커스텀 프로퍼티를 참조합니다. 색·간격을 바꿀 땐 토큰만 수정하면 전 화면에 반영됩니다.
- **명명**: BEM-lite — `block__elem`, `block--mod`, 상태는 `is-*` (`is-on`, `is-active`).
- **아이콘**: [Phosphor Icons](https://phosphoricons.com) (MIT) regular weight. 각 페이지 상단에 그 페이지가 쓰는 아이콘만 인라인 스프라이트로 포함하고 `<svg class="ph"><use href="#ph-이름"/></svg>`로 사용합니다. CDN을 쓰지 않는 이유: 오프라인·보안 정책 환경에서도 파일 하나로 동작해야 하기 때문.
- **상태 표현**: 인터랙션 상태는 클래스로만 표시해 둡니다 (예: 확인 모드 켜짐 = `.chip.is-on`). 동작 연결은 개발 단계에서.
