설계안 v2 — 좌 채팅 · 우 기획안 (명세 §2.5)NEW 07-22 17:33

규격은 고정, JSON이 값만 채운다. 좌 채팅 질의 · 우 기획안의 2단 스튜디오 — 질의 중 우측은 빈 규격(스켈레톤)이고, 티키타카가 끝나면 문서 전체가 한 번에 채워진다. 수정은 채팅으로, 확정하면 생성 시작.

핵심 원칙
  • 일괄 바인딩 — 질의가 끝나면 에이전트 JSON 1회 도착으로 문서 전체가 채워진다. 중간중간 채우는 연출 없음 → 프런트는 고정 템플릿에 값만 꽂으면 되므로 구현·응답이 빠르다
  • 레이아웃 = B2 스튜디오와 동일 2단 — 좌 채팅 · 리사이저 · 우 패널
  • 질의 중 = 스켈레톤 — 섹션 제목은 실제, 값 자리만 회색 바 ("무엇이 만들어질지" 예고편)
  • 양식 = 밑줄 섹션 제목 + 좌 라벨·우 내용 행 (2026-07-23 확정) — 위에서 아래로 이어지는 문서형. 왼쪽 라벨만 훑어도 구조가 읽힌다
  • 문서에 이모지 금지 — 플랫하게. 완성 후 접힘 재선택으로 답을 바꾸면 해당 슬롯만 즉시 갱신
흐름 — 여기까지가 이 화면의 범위

요청 → 진행 방식 선택 → 기능 질의 → 의견 확인 → 기획안 바인딩 → 디자인 질의 → 디자인 탭 → 확정. 기능 질의가 끝나면 개방형 의견 확인("빠진 게 있나요?")을 한 번 거쳐 기획안을 채우고, 이어지는 디자인 질의 결과는 레일의 디자인 탭(완성 시 점 배지)에 담긴다. 맡기기 모드는 의견 확인 없이 추천값으로 기획안·디자인까지 일괄. 완성되면 "이대로 만들기 시작 vs 와이어프레임으로 먼저 볼래요"로 갈라진다 — 생성 진행은 별도 화면(b2).

B1 아이디어 전송 한 문장 요청
→
AI ① 기능 질의 N개 답 → 우측 ①에 "대화로 확정" 행
→
AI ② 디자인 질의 답 → 우측 ② 디자인 방향 카드
→
AI → 확인 대기 🔒 기획안 완성 화면 흐름·저장 정보 공개 + 확정
우측 패널(아티팩트) 규칙

스켈레톤 → 일괄 바인딩 → 슬롯 갱신, 세 상태뿐이다.

  • 질의 중 = 스켈레톤 — 섹션 제목은 실제, 값 자리는 회색 바 · 상단 바 문구 "질문이 끝나면 여기에 채워져요"
  • 완료 = 일괄 바인딩 — 채워진 문서로 교체(스크롤 맨 위로) · 배지 "대기 중"→"완성" · 문구 "확인해 주세요" · 확정은 채팅의 경로 선택이 담당(문서 하단 확정 버튼 없음 — 2026-07-24 제거)
  • 완성 후 = 슬롯 갱신만 — 접힘 재선택으로 답을 바꾸면 해당 슬롯만 즉시 갱신
  • 행 단위 수정 (v3) — 항목 줄에 호버하면 하이라이트 + "이 부분 수정하시겠어요?" 버튼, 클릭 시 컴포저에 "섹션 · 항목" 칩 · 💬 아이콘 폐기
  • 레일 = [기획안 | 디자인] (v3) — 디자인 브리프는 전용 탭으로 분리(완성 시 점 배지) · 기획안 왼쪽에 플로팅 목차(스크롤 스파이 + 클릭 점프) · 개요는 한 줄 정의·목적·사용자·플랫폼 4행
  • 배경은 미리보기 페인과 같은 rgb(249 250 251) — 스튜디오와 시각 연속성 · 리사이저는 b2와 동일(기본 채팅 28%, 300~720px) · 900px 이하 1단
질의 규칙 (채팅 쪽)

화면·데이터·스타일 토큰이 실제로 갈리는 것만 묻는다. 기능 질의 개수는 에이전트가 요청을 분석해 정하고(시연은 2개), 디자인은 분위기·글씨 2개 안팎으로 수렴한다.

  • 되묻기 카드는 제목+설명문(§2.4) · 옵션 2~3 + AI 추천 · 디자인 옵션에 톤 견본 · "잘 모르겠어요" = 추천 적용
  • 질의 중 컴포저는 타이핑 가능·전송만 잠금 → 기획안 완성 시 해제(수정 = 채팅, 별도 수정 버튼 없음)
  • 기획안 문서에 이모지 금지 — 태그·견본·타이포만으로 구분 (2026-07-22 확정)
기획안 문서 ↔ 에이전트 JSON (Manifest PRD 절충 — 2026-07-22)

문서의 모든 슬롯은 JSON 필드 하나와 1:1 대응이다 — 화면 전용 데이터를 만들지 않는다. Manifest의 기획서 항목은 전부 수용하고, 기존 JSON에 없는 항목(⊕)은 확장 필드로 제안한다 (프롬프트 원문 문서 참조).
