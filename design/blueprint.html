핵심 원칙

화면에 보이는 모든 정보는 AI 기획 산출물에서 온다 — 화면 전용 데이터를 만들지 않는다. 질문에 답해야 다음 질문이 나타난다(순차 진행). 필드 단위 계약 매핑은 명세 §2.5가 단일 출처.

단계 흐름
사용자 · B1 아이디어 전송 컴포저 + 첨부·작업유형
→
AI → 승인 대기 🔒 ① 기획안 카드 기획 초안 생성 · 수정 요청 시 부분 재생성
→
사용자 · 원탭 ② 분위기 4종 카드 · 추천 선선택
→
사용자 · 토글 ③ 기능 기본 2개 on · 확정 시 접힘
→
수시 발생 ④ 되묻기 모호한 요청 → 옵션으로 좁힘
→
전이 · building_ui ⑤ 만들기 시작 확정 계약 스냅샷 → B2
단계별 정보 · 동작
단계보여주는 정보사용자 행동스킵 시
① 기획안서비스명 · 소개 · 해결 문제 · 핵심 기능 · 화면 N개 · 데이터 저장소 · 타겟[좋아요, 다음으로] 승인 / [수정 요청] → 채팅으로 고쳐 말함차단 — 승인 대기
② 분위기스타일 4종 카드 + 직접 입력 (추천 선선택)원탭 선택 → 접힘추천안 적용
③ 기능빈출 기능 칩 (기본 2개 on, 외부 절차엔 '출시 후 연결')칩 토글 → [확정] → 접힘기본 on 유지
④ 되묻기모호한 요청 → 옵션 2~4 (AI 추천 표시) + 기타원탭 즉답 / 기타 입력 → 접힘AI 추천 적용
⑤ 시작확정 요약 + [만들기 시작]클릭 → B2 진입—
질의 UX 5원칙
  • 한 번에 하나 — 여러 답이 필요하면 대화형 스텝으로
  • 기본값 선제 선택 — 무응답이면 기본값으로 진행 (스킵 ≠ 실패)
  • 선택지 최대 4개 + 직접 입력 — 자유 답변은 카드의 '기타' 입력으로
  • 질의 활성·작업 중엔 전송 잠금 (타이핑·미리 써두기는 가능) — 답을 먼저 받는다
  • 답하면 접힘 — "라벨: 값 ✓" 요약이 채팅에 남고, 클릭 시 재선택
구현 규칙
  • 수정 요청 → 기획 에이전트가 해당 부분만 재생성, 답한 항목은 보존
  • ① 승인이 생성 시작의 트리거 — 이 지점 외의 차단 개입은 없음
  • 되묻기(④)는 채팅 공통 패턴 — 운영 중(b2)에서도 동일 컴포넌트
  • 자율 모드: 비차단 질문 생략, 완료 보고에 "AI가 대신 고른 것" 목록
